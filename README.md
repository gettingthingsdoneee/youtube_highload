## 2. Функциональность MVP

1. Просмотр видео
2. Возможность выложить видео
3. Комментарии под видео
4. Лайки/дизлайки под видео
5. Поиск по видео/каналам 
6. Просмотр рекомендаций
7. Просмотр канала (списка видео канала)

 ## 3. Ключевые продуктовые решения
1. **Глобальная CDN**: Видеоконтент доставляется через распределенную сеть серверов (CDN). Популярные видео кэшируются на периферийных узлах (PoPs), максимально близких к зрителю, что минимизирует задержки и нагрузку на магистральные каналы.

2. **Адаптивный стриминг (ABR)**: Видео хранится в нескольких битрейтах (HLS/DASH). Плеер на клиенте динамически выбирает оптимальное качество, обеспечивая плавное воспроизведение при изменениях пропускной способности сети.

3. **Асинхронный пайплайн обработки видео**: Загруженное видео помещается в очередь сообщений. Специализированные воркеры (кодировщики) асинхронно конвертируют его во все необходимые форматы, генерируют миниатюры и субтитры, не блокируя работу пользователя с приложением.

4. **Микросервисная архитектура и специализированные хранилища**: Система разделена на независимые микросервисы (рекомендации, поиск, комментарии). Для каждого сервиса используется своя база данных, оптимизированная под его задачи: Elasticsearch для поиска, BigTable/Cassandra для истории комментариев, Redis для кэша и счетчиков лайков/просмотров.
